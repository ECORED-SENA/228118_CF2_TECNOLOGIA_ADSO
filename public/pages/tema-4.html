4. Validación de requisitos

4.1 Comprobaciones

La validación de requerimientos es el proceso de comprobar que los requisitos que el cliente solicita se especifique de forma correcta. Se debe realizar una serie de comprobaciones sobre el documento de requisitos. En el capítulo 4, página 110 se explican los diferentes tipos de comprobaciones:

4.2 Técnicas

Teniendo en cuenta la necesidad que desde los inicios del proceso de software, se minimicen los errores, equivocaciones y malentendidos que se pueden presentar durante el desarrollo de software, lo cual puede ocasionar grandes costos en el producto final, para ello es importante aplicar unas técnicas para validación de requisitos que se encuentran en el capítulo 4, página 111.
